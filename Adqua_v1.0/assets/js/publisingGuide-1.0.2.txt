책상위에 프린트해서 붙여놓고, 프로젝트 진행시 수시로 간단히 체크 가능하도록 함이 목표입니다.

1. Client Orientation & Kick off Meeting 단계
	1) 일정과 난이도에 따른 인력배치( 팀장 또는 선임 참여 )
	2) 프로젝트 환경 체크

2. Ideation 회의 / 스토리보드 회의 단계
	1) Tech팀 담당자 선정 및 일정확인 ( 앞단계에 미리 결정되어있을 수 있슴 )
	2) UX/UI 레퍼런스사이트 수집/제공
	3) 기술구현 가능여부, 진행 상 제한사항 체크 ( 클라이언트 내부가이드, 접근성 이슈, 크로스 브라우징 범위 등 )
	4) 이 프로젝트를 사용자입장에서 봤을 때 브랜드에 호감이 생기고 참여하고 싶을까?

3. 스토리보드 컨펌 및 디자인 시안 단계
	1) 스토리보드 컨텐츠 분석
	2) 웹/모바일 형태로 전환가능한 디자인 시안인지 확인
	3) 기술구현 리소스 수집, 기능 테스트

4. 디자인 시안컨펌 및 디자인 수행 단계
	1) 디자인 시안 컨텐츠 분석 기능관련 테스트페이지 제작, HTML 문서 구조설계 ( 인코딩, DTD 정의 등 )
	2) 디자인 진행 시 꼭 한번이상 디자이너 자리에서 확인
	3) 일정체크 및 조율(휴일이나 다른 프로젝트 이슈가 있을 시 미리 인수인계)

5. 디자인 완료 단계
	1) 폴더/파일 네이밍, CSS, JS 작성, JavaScript 작업 및 가이드 마련
	2) 인터럭션 관련 디자인 수정 및 이미지, 폰트 등 소스 요청
	3) 작업 후 소스파일이나 URL 완료 메일

6. 개발 진행 단계
	1) 크로스 디바이스, 브라우징 테스트, 개발연동 관련 수정 및 메일공유
	2) 개발 진행 시 꼭 한번이상 개발자 자리에서 확인
	3) 브라우저별 렌더링 화면과 디자인 비교, 웹접근성/표준화 검사(필요 시)

7. 테스트 단계
	1) 디자인 및 수정 요청 적용, 소스정리
	2) 테스트 오류 수정 및 메일공유
	3) 산출물 최종 확인 브라우저 테스트 및 동작 테스트

8. 오픈 후 리뷰 단계
	1) 새롭게 시도해본 방식 정리, 라이브러리/참고문서화
	2) 브랜드, 협업 간 특징 이슈리스트 정리
	3) 부족했던 부분, 더 발전되어야 할 부분 체크
